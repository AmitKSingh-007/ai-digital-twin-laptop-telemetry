# AI Digital Twin for Laptop Telemetry

# API Design Documentation

## Overview

This document defines all backend APIs exposed by the Digital Twin platform.

The APIs are grouped into four categories:

1. Telemetry APIs
2. Twin State APIs
3. Simulation APIs
4. AI Assistant APIs

All responses use JSON.

Base URL:

```text
http://localhost:5000
```

---

# API 1 — Get Telemetry History

## Endpoint

```http
GET /api/telemetry
```

---

## Purpose

Returns the complete telemetry history generated by the telemetry engine.

Used by:

* Dashboard charts
* Trend analysis
* Historical visualization

---

## Sample Request

```http
GET /api/telemetry
```

---

## Sample Response

```json
[
  {
    "timestamp": "2026-06-18T10:00:00Z",
    "cpu_utilization_pct": 35,
    "cpu_temperature_c": 58,
    "fan_speed_rpm": 1600,
    "battery_drain_w": 6.5
  },
  {
    "timestamp": "2026-06-18T10:01:00Z",
    "cpu_utilization_pct": 42,
    "cpu_temperature_c": 60,
    "fan_speed_rpm": 1600,
    "battery_drain_w": 7.1
  }
]
```

# API 6 — Run Simulation

## Endpoint

```http
POST /api/simulate
```

---

## Purpose

Runs a What-If simulation.

---

## Supported Scenarios

```text
fan_failure
desert_heat
heavy_compile
```

---

## Request

```json
{
  "scenario": "fan_failure"
}
```

---

## Sample Response

```json
{
  "scenario": "fan_failure",

  "result": {
    "predictedTemperature": 98,
    "thermalThrottling": true,
    "performanceLossPct": 45
  }
}
```

# API 8 — AI Chat

## Endpoint

```http
POST /api/chat
```

---

## Purpose

Allows users to interact with the Digital Twin using natural language.

---

## Request

```json
{
  "message": "Why is CPU temperature increasing?"
}
```

---

## Backend Processing

The backend gathers:

1. Twin State
2. Predictions
3. Anomalies
4. Simulation Context

and sends them to Gemini.

---

## Sample Response

```json
{
  "reply": "Observation: CPU temperature increased from 66°C to 74°C.

Correlation: Increased CPU load caused higher power consumption, leading to thermal buildup.

Prediction: Temperature may exceed 80°C within 15 minutes.

Recommendation: Reduce workload or improve cooling."
}
```

# Conclusion

This API architecture separates telemetry collection, twin-state management, prediction, anomaly detection, simulation, and AI reasoning into independent modules. This modular design allows all team members to develop and test their components independently while maintaining a consistent contract between frontend and backend.
